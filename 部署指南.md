## 一、后端部署（阿里云服务器）

### 1.1 环境准备

```bash
# Ubuntu 环境
sudo apt update && sudo apt upgrade -y
sudo apt install -y python3 python3-pip python3-venv nginx mysql-server

# CentOS 环境
sudo yum update -y
sudo yum install -y python3 python3-pip nginx mysql-server
```

### 1.2 MySQL 配置

```bash
# 启动 MySQL 服务
sudo systemctl start mysqld        # CentOS
sudo systemctl start mysql         # Ubuntu
sudo systemctl enable mysqld

# 登录 MySQL（首次使用默认密码，查看日志获取临时密码）
sudo grep 'temporary password' /var/log/mysqld.log
mysql -u root -p
```

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS microshare
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- 创建专用用户（可选）
CREATE USER 'microshare'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON microshare.* TO 'microshare'@'localhost';
FLUSH PRIVILEGES;

-- 确认数据库字符集
SHOW VARIABLES LIKE 'character_set_database';
```

### 1.3 上传项目文件（SCP）

在本地开发机（Windows）上使用 scp 上传后端文件：

```powershell
# Windows PowerShell 上传整个 backend 目录
scp -r F:\MicroShare\backend root@8.137.186.43:/opt/microshare/

# 只上传核心文件（不含虚拟环境等）
scp F:\MicroShare\backend\*.py root@8.137.186.43:/opt/microshare/
scp F:\MicroShare\backend\requirements.txt root@8.137.186.43:/opt/microshare/

# 如果需要上传上传目录
scp -r F:\MicroShare\backend\uploads root@8.137.186.43:/opt/microshare/
```

### 1.4 安装 Python 依赖

```bash
cd /opt/microshare

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 如 requirements.txt 缺失，手动安装
pip install flask flask-cors pymysql cryptography
```

### 1.5 初始化数据库

```bash
cd /opt/microshare
source venv/bin/activate

# 确保 config.py 中 MySQL 配置正确
# 关键参数：host=127.0.0.1, port=3306, user=root, password=你的密码, database=microshare

python init_db.py
# 输出示例：数据库初始化完成，已创建 7 张表
```

### 1.6 配置 systemd 服务

创建 Flask API 服务：

```bash
sudo vi /etc/systemd/system/microshare-api.service
```

```ini
[Unit]
Description=MicroShare Flask API Server
After=network.target mysql.service
Wants=mysql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/microshare
Environment=PATH=/opt/microshare/venv/bin
ExecStart=/opt/microshare/venv/bin/python /opt/microshare/app.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

创建 Socket 推送服务：

```bash
sudo vi /etc/systemd/system/microshare-socket.service
```

```ini
[Unit]
Description=MicroShare Socket Push Server
After=network.target mysql.service
Wants=mysql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/microshare
Environment=PATH=/opt/microshare/venv/bin
ExecStart=/opt/microshare/venv/bin/python /opt/microshare/socket_server.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable microshare-api microshare-socket
sudo systemctl start microshare-api microshare-socket

# 检查运行状态
sudo systemctl status microshare-api
sudo systemctl status microshare-socket

# 查看日志
sudo journalctl -u microshare-api -f
sudo journalctl -u microshare-socket -f
```

### 1.7 Nginx 反向代理

```bash
sudo vi /etc/nginx/conf.d/microshare.conf
```

```nginx
server {
    listen 80;
    server_name 8.137.186.43;

    # Flask API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 上传文件访问
    location /uploads/ {
        alias /opt/microshare/uploads/;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # 根路径
    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 大文件上传支持
    client_max_body_size 20M;
}
```

```bash
# 测试配置并重启 Nginx
sudo nginx -t
sudo systemctl restart nginx
sudo systemctl enable nginx

# 配置防火墙放行 80 端口
sudo firewall-cmd --permanent --add-service=http    # CentOS
sudo firewall-cmd --reload
# 或
sudo ufw allow 80/tcp                                # Ubuntu
```

## 二、Android 构建

### 2.1 构建 Release APK

1. 用 Android Studio 打开 `F:\MicroShare\android`
2. 等待 Gradle Sync 完成
3. 菜单栏选择 **Build > Generate Signed Bundle / APK**
4. 选择 **APK**，点击 Next
5. 选择或创建密钥库（Key store path），填写密码和别名
6. 选择 **release** 构建变体
7. 签名版本勾选 **V1 (Jar Signature)** 和 **V2 (Full APK Signature)**
8. 点击 Finish，等待编译完成
9. 生成的 APK 位于：
   ```
   android\app\build\outputs\apk\release\app-release.apk
   ```

### 2.2 构建 Debug APK（直接安装测试）

```bash
# 方式一：Android Studio 直接点击 ▶️ Run（自动安装到连接的设备）

# 方式二：使用 Gradle 命令行构建
cd F:\MicroShare\android
./gradlew assembleDebug

# 生成的 Debug APK 位于：
# android\app\build\outputs\apk\debug\app-debug.apk
```

### 2.3 安装到设备

```bash
# 连接手机（开启 USB 调试），执行：
adb install -r android\app\build\outputs\apk\debug\app-debug.apk

# 或直接将 APK 文件传到手机上点击安装
```

### 2.4 配置服务器地址

修改 `android/app/src/main/java/com/microshare/config/AppConfig.kt`：

```kotlin
object AppConfig {
    // 阿里云服务器地址
    const val BASE_URL = "http://8.137.186.43"
    const val SOCKET_HOST = "8.137.186.43"
    const val SOCKET_PORT = 9000

    // 模拟器测试时改为：
    // const val BASE_URL = "http://10.0.2.2"
    // const val SOCKET_HOST = "10.0.2.2"
```

## 四、常见问题

### Q1: 防火墙导致无法访问

**现象**：客户端连接超时，`curl` 请求无响应。

**解决**：

```bash
# 查看防火墙状态
sudo systemctl status firewalld    # CentOS
sudo ufw status                     # Ubuntu

# 放行必要端口
sudo firewall-cmd --permanent --add-port=5000/tcp   # Flask API
sudo firewall-cmd --permanent --add-port=9000/tcp   # Socket
sudo firewall-cmd --permanent --add-service=http    # Nginx 80
sudo firewall-cmd --reload

# 阿里云安全组还需要在控制台添加入方向规则：
# - 端口 5000（Flask API，调试用）
# - 端口 9000（Socket 推送）
# - 端口 80（Nginx 反向代理）
# - 端口 443（HTTPS，如需）
```

### Q2: Socket 服务无法连接

**现象**：消息通知功能不可用，客户端提示"无法连接到推送服务"。

**排查步骤**：

```bash
# 1. 检查 Socket 服务是否运行
sudo systemctl status microshare-socket
sudo journalctl -u microshare-socket -n 20

# 2. 检查端口监听
sudo netstat -tlnp | grep 9000

# 3. 手动测试 Socket
python -c "import socket; s=socket.socket(); s.connect(('127.0.0.1', 9000)); print('OK'); s.close()"

# 4. 重启服务
sudo systemctl restart microshare-socket

# 5. 如果还是不行，检查防火墙和阿里云安全组是否放行了 9000 端口
```

### Q3: 图片上传后不显示（404）

**现象**：发帖带图片后，图片链接返回 404。

**排查步骤**：

```bash
# 1. 确认 uploads 目录存在且权限正确
ls -la /opt/microshare/uploads/
sudo chmod -R 755 /opt/microshare/uploads/

# 2. 确认 Nginx 配置中 /uploads/ 的位置指向正确
sudo nginx -t
# 检查 alias 路径是否与实际 uploads 目录一致

# 3. 查看 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log

# 4. 直接访问文件测试
curl -I http://8.137.186.43/uploads/xxx.jpg

# 5. 检查 config.py 中的 SERVER_HOST 配置
# SERVER_HOST = "http://8.137.186.43"
```

### Q4: Android 编译报错

**现象**：Gradle Sync 失败或编译时报错。

**常见原因与解决**：

| 错误类型 | 原因 | 解决方案 |
|----------|------|----------|
| `Gradle sync failed` | Gradle 版本不匹配 | 检查 `gradle-wrapper.properties` 中的 distributionUrl，确保与本地 Gradle 版本兼容 |
| `Unresolved reference: xxx` | 依赖下载失败 | 检查网络连接，配置阿里云 Maven 镜像仓库加速 |
| `Android SDK not found` | SDK 路径未配置 | 在 Android Studio 中设置 SDK 路径：File > Settings > Appearance & Behavior > System Settings > Android SDK |
| `Build output path is too long` | Windows 路径过长 | 将项目移动到盘符根目录（如 `D:\microshare`） |
| `INSTALL_FAILED_UPDATE_INCOMPATIBLE` | 已安装签名不同的版本 | 卸载设备上的旧版 APK 后重新安装 |
| `connect refused (10.0.2.2:5000)` | 模拟器无法连接后端 | 确认后端服务已启动；Windows 本地开发时，将 AppConfig.kt 中的地址改为 `10.0.2.2`（模拟器映射宿主机 localhost） |

### Q5: MySQL 连接失败

**现象**：启动 Flask 时提示 `pymysql.err.OperationalError`。

**排查步骤**：

```bash
# 1. 确认 MySQL 服务运行中
sudo systemctl status mysql

# 2. 确认 config.py 中的数据库凭据正确
cat /opt/microshare/config.py

# 3. 测试 MySQL 连接
mysql -u root -p -e "SELECT 1"

# 4. 确认数据库已创建
mysql -u root -p -e "SHOW DATABASES;"

# 5. 检查 MySQL 是否监听 127.0.0.1
sudo netstat -tlnp | grep 3306
# 确认 bind-address 不是 127.0.0.1 以外的地址
```

### Q6: Python 依赖安装失败

**现象**：`pip install` 时报错。

```bash
# 使用国内镜像源加速
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 如果 cryptography 编译失败，安装系统依赖
sudo apt install -y build-essential libssl-dev libffi-dev python3-dev
```
