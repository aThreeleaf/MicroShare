## 一、项目概述

微分享社区是一款融合社交媒体与物联网传感器技术的移动社交应用。它借鉴小红书的内容分享模式，支持用户发布图文帖子、互动交流（点赞/评论/收藏/关注）、实时消息推送，并创新性地集成手机加速度传感器计步与运动排行榜功能，将线上社交与线下运动有机结合。

**项目定位**：社交博客论坛 + 物联网传感器融合应用

**核心特色**：
- 内容分享：图文帖子流、话题标签、分页加载
- 社交互动：点赞、评论、收藏、关注、实时通知推送
- 运动传感：加速度计步、运动数据记录、每日排行榜

### 3.3 systemd 服务配置

**microshare-api.service**（Flask API）：
- WorkingDirectory: `/opt/microshare`
- ExecStart: `venv/bin/python app.py`
- Restart: `on-failure`, RestartSec: 5s
- After: `network.target mysql.service`

**microshare-socket.service**（Socket 推送）：
- WorkingDirectory: `/opt/microshare`
- ExecStart: `venv/bin/python socket_server.py`
- Restart: `on-failure`, RestartSec: 5s
- After: `network.target mysql.service`

## 七、考核技术覆盖

| 考核点 | 实现方式 | 状态 |
|--------|----------|------|
| 网络通信（HTTP） | Flask RESTful API + OkHttp 客户端，24 个 REST 接口 | 已覆盖 |
| 网络通信（Socket） | 独立 TCP Socket 服务器（端口 9000），长连接管理 + 心跳检测 + 实时推送 | 已覆盖 |
| 数据库服务器（MySQL） | MySQL 8.0 远程数据库，7 张 InnoDB 表，外键约束，联合索引 | 已覆盖 |
| 传感器（加速度计步） | `StepDetector.kt` 基于 `TYPE_ACCELEROMETER` 实现步数检测算法 | 已覆盖 |
| 多协程 | Kotlin Coroutines 管理异步任务（网络请求、Socket 读写、传感器数据采集） | 已覆盖 |

**技术亮点说明**：

1. **HTTP + Socket 双通道通信**：HTTP 用于常规 API 请求（CRUD 操作），Socket 用于实时推送（点赞/评论/关注通知），两者可同时独立工作。
2. **MySQL 数据库服务器**：部署在阿里云 ECS 本地，7 张表通过外键建立完整的关系约束，支持复杂查询（如用户帖子数统计、关注关系判断）。
3. **加速度传感器计步**：通过 `SensorManager.TYPE_ACCELEROMETER` 采集三轴加速度数据，使用波峰检测算法识别步伐，并在前台 Service 中持续运行。
4. **多协程并发**：使用 `kotlinx.coroutines` 管理网络 IO、Socket 通信、传感器数据读取等多个并发任务，避免主线程阻塞。
